Detailed Description

template<class T>
class CircularBuffer< T >

Generic circular buffer.

This is loosely modelled on a subset of the STL containers. It may eventually get iterators, etc., but presently has only the interface needed to satisfy the union of client code requirements. T must support assignment sensibly, as that is how insertions are done. The overflow policy is silent overwriting: clients are responsible for avoiding this if they don't want it.

Attention
Note that the head/front is the writing end, and the tail/back is the reading end: this is the ordinary nomenclature for circular buffers, but is opposite to STL usage-we have to disappoint someone.
